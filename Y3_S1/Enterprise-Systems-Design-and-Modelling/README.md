# 🏢 Enterprise Systems Design & Modelling (WBL)

Welcome to the repository for my **Enterprise Systems Design and Modelling (WBL)** course at Universiti Teknologi Malaysia (UTM)! This repository documents a comprehensive analysis and development lifecycle aimed at modernizing university enterprise architectures.

---

## 📂 Project & Assignment Breakdown

### 🔹 Enterprise Architecture Project (UTHM Case Study)
* **File:** `/ESDM_Project_Group2_Final.pdf`
* **Summary:** A comprehensive report analyzing UTHM's legacy IT environment to design a unified, target "TO-BE" Enterprise Architecture. Using the TOGAF ADM Framework, we mapped out a 3-tier Smart Campus model featuring a Single Source of Truth (SSOT), centralized API gateway, and proactive security management (SIEM) to replace fragmented campus platforms.

### 🔹 SAP Hands-on Labs & Technology Implementations
* **File:** `/SAP_Report_Farra_Section1.pdf`
* **Summary:** 
  * **Lab 1:** Built a mobile-optimized **Sales Order Workflow Application** in SAP Build Apps using zero-code visual scripting, dynamic formulas, and automated CRUD pages integrated with SAP S/4HANA backend services.
  * **Lab 2:** Developed a functional smartphone **Barcode Scanner Application** that utilizes native mobile services to fetch and map live JSON data from a public REST API (OpenFoodFacts) onto the app's user interface.

### 🔹 Industrial Engagements & Comparative Research
* **Industrial Visit Report** (`/Industrial_Visit_UTHM_Summary.pdf`): Summarizes a physical study of the UTHM Data Center, focusing on Tier 3 high-availability infrastructure, redundant power routes, and hot/cold aisle containment cooling systems.
* **Industrial Seminar** (`/PPG_Industrial_Talk_Summary.pdf`): Covers an industry insights session with PPG global leads regarding large-scale enterprise ERP deployments (SAP S/4HANA) spanning standard business workflows like Order-to-Cash and Procure-to-Pay.
* **Research Paper** (`/Cloud_vs_OnPremise_ERP_Report.pdf`): A comparative technical paper evaluating the structural trade-offs, architecture styles, and business cases for Cloud-Based ERP vs. Traditional On-Premise ERP systems.

## 💡 Course Reflection & Key Takeaways

1. **Framework-Driven Design (TOGAF):** Learned how to align software design with core business strategy by mapping systems across Business, Data, Application, and Technology layers.
2. **Eliminating Silos:** Gained experience designing centralized, unified systems (SSOT) to eliminate operational fragmentation and data redundancy.
3. **Low-Code/Cloud Integration:** Developed practical skills in rapid prototyping and integrating cloud-native services (SAP Build Apps & BTP) with backend ERP systems and public APIs.
